# SciQuiz

เว็บเกมตอบคำถามวิทยาศาสตร์สำหรับนักเรียน รวมคำถาม 4 ชุด ชุดละ 80 ข้อ รวม 320 ข้อ โดยเก็บคำถามเดิมครบและเพิ่มคำถามวิเคราะห์ชุดละ 30 ข้อ

- ป.1–2 พืชและสัตว์
- ป.3–5 อวัยวะในร่างกายและระบบย่อยอาหาร
- ป.6–ม.1 ระบบต่าง ๆ ในร่างกาย
- ม.2–ม.3 ภัยพิบัติและการรับมือ

## ความสามารถ

- แต่ละเกมใช้ 50 ข้อในสัดส่วนคงที่: ง่าย 15 ข้อ (30%) ปานกลาง 15 ข้อ (30%) และยาก 20 ข้อ (40%) โดยไม่ซ้ำข้อ
- ข้อ 1–5 เป็นข้อง่ายเพื่อเริ่มเกม จากข้อ 6–50 จึงสุ่มคละระดับและไม่ใช้ข้อเดิมซ้ำในรอบนั้น
- โจทย์วิเคราะห์ชุดเสริมปรับให้สั้น อ่านจบไว และทบทวนความสัมพันธ์ระหว่างโจทย์–ตัวเลือก–เฉลยแล้ว
- คำถามเสริมระดับปานกลาง–ยาก 120 ข้อใช้สถานการณ์ ผลการสังเกต และการเปรียบเทียบเพื่อวิเคราะห์ ช้อยส์ทั้งสองฝั่งเกี่ยวข้องกับโจทย์และมีความยาวใกล้กัน แต่มีคำตอบซึ่งถูกต้องและครบที่สุดเพียงข้อเดียว (ชุดคำถาม v5)
- สุ่มตำแหน่งคำตอบซ้าย–ขวาใหม่ทุกเกม พร้อมควบคุมไม่ให้เกิดแพตเทิร์นที่เดาง่าย
- แสดงรูปประกอบ ตัวเลือกซ้าย–ขวา ตัวจับเวลา และคะแนน
- เลือกเวลาได้ 10, 15 หรือ 20 วินาทีต่อข้อ โดยค่าแนะนำคือ 15 วินาที และลดช่วงรอเฉลยเพื่อให้เกมต่อเนื่องขึ้น
- หน้าจอผู้ดูแลสำหรับเพิ่ม แก้ไข และลบชุดคำถาม
- นำเข้าไฟล์ Excel ตามหัวตารางเดิมได้
- Responsive สำหรับโทรศัพท์และ iPad ทั้งแนวตั้ง–แนวนอน รวมถึง MacBook และโน้ตบุ๊ก
- บันทึกข้อมูลในเบราว์เซอร์อัตโนมัติ โดยไม่ต้องมีเซิร์ฟเวอร์
- โหมด AV เปิดกล้อง นับใบหน้าเป็นหลักและใช้จุดศีรษะเป็นระบบสำรอง มีการติดตามตำแหน่งเพื่อลดการนับซ้ำและลดตัวเลขกระพริบ
- โหมด AV ตัดสินถูก–ผิดตามฝั่งคำตอบจริง ผู้เล่นที่ยืนฝั่งถูกจะถูกนับว่าตอบถูก โดยไม่ใช้ฝั่งที่มีคนมากกว่าเป็นคำตอบรวม

## วิธีใช้โหมด AV กล้อง

1. เปิดเว็บด้วย Chrome หรือ Edge ผ่านลิงก์ HTTPS ของ GitHub Pages
2. เลือกชุดคำถาม แล้วเลือก `โหมด AV กล้อง`
3. กด `เริ่มเล่นเกม` และอนุญาตให้เว็บไซต์ใช้กล้อง
4. วางกล้องให้เห็นใบหน้าและศีรษะของนักเรียนชัดเจน ให้เด็กหันหน้าเข้ากล้องและเว้นระยะไม่บังกัน
5. ให้เด็กยืนพ้นแถบกลางไปทางซ้ายหรือขวา ระบบจะแสดงหมายเลขกำกับและนับจำนวนแต่ละฝั่งแบบเรียลไทม์
6. เมื่อหมดเวลา ฝั่งคำตอบถูกจะแสดงสีเขียว ฝั่งผิดจะแสดงสีแดง พร้อมจำนวนผู้ตอบถูก ผู้ตอบผิด และผู้ที่ยังยืนกลางภาพ

ภาพจากกล้องประมวลผลภายในเบราว์เซอร์ ไม่บันทึกเป็นไฟล์และไม่ส่งเข้าฐานข้อมูล จำนวนคนสองฝั่งจะไม่ถูกนำมาโหวตหาคำตอบรวม ควรจัดแสงด้านหน้าให้เพียงพอ วางกล้องให้นิ่ง และไม่ให้ใบหน้าซ้อนกันเพื่อเพิ่มความแม่นยำ

## การแสดงผลตามอุปกรณ์

- โทรศัพท์แนวตั้ง: คำถามอยู่ด้านบน และแสดงช้อยส์ซ้าย–ขวาพร้อมกันด้านล่าง
- โทรศัพท์แนวนอน: จัดเป็นฝั่งซ้าย–คำถาม–ฝั่งขวา เพื่อใช้พื้นที่แนวนอนเต็มจอ
- iPad: ปรับขนาดคอลัมน์ รูปคำถาม ปุ่ม และตัวจับเวลาให้เหมาะกับทั้งสองแนว
- MacBook/โน้ตบุ๊ก: รักษารูปแบบสามคอลัมน์และย่อส่วนอัตโนมัติเมื่อหน้าจอมีความสูงน้อย
- รองรับพื้นที่ขอบจอของ iPhone/iPad และความสูงหน้าจอแบบ Dynamic Viewport

## การตั้งค่า

ไฟล์ `config.js` กำหนด PIN เริ่มต้นเป็น `0000` และใช้การบันทึกในเบราว์เซอร์ หากต้องการให้ทุกเครื่องใช้ข้อมูลร่วมกัน สามารถใส่ Firebase Web Config และกำหนด Firestore Security Rules ที่เหมาะสมได้

> PIN ในเว็บแบบ Static เป็นเพียงตัวกันการกดโดยไม่ตั้งใจ เพราะไฟล์ JavaScript ของเว็บไซต์สาธารณะสามารถเปิดอ่านได้ ไม่ควรใช้แทนระบบยืนยันตัวตนจริง

## เผยแพร่ด้วย GitHub Pages

1. อัปโหลดไฟล์ทั้งหมดขึ้น Repository โดยใช้สาขา `main`
2. เปิด `Settings` → `Pages`
3. เลือก Source เป็น `GitHub Actions`
4. เปิดแท็บ `Actions` และรอ Workflow `Deploy SciQuiz to GitHub Pages` ทำงานเสร็จ
